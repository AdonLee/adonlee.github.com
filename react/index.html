Advantage:

  1. 模块化，代码解耦，组件复用
  2. 虚拟DOM，不直接操作DOM，只对改变的部分做更新，延迟批处理，提高了渲染性能
  3. 单向数据流，通过更新父组件的状态（setState方法）便可触发子组建的UI更新，开发者仅仅需要考虑整体界面的DOM构建，不再需要关心局部更新。
  4. 主要做为MVC架构中的V层，可作为一个新特征在原有项目中使用
  5. React Native可用于编写原生应用
  6. 代码可预编译化，减少代码量，提高可阅读性,
    碉堡的开发栈: ES6 + JSX + SASS/LESS
  7. 支持后端Nodejs渲染，减轻浏览器DOM支持, 然并卵
Tips

单一功能原则：理想情况下一个组件应该只做一件事，当组件逐渐变大变复杂就需要拆分成更小的子组件

单向数据流： React 中数据是通过prop属性沿着组件树从上到下单向流动的。根据state来重新渲染组件，state维护在所有基于此state的组件的共同父组件, 父组件通过传递更新state的回调给子组件来实现数据的反向流动，这样保证了state的唯一性.也可通过linkstate插件实现类似Angular的双向数据绑定
